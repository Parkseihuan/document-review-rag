# 문서관리규정

이 파일은 기안서 작성 시 자동으로 검토할 규정들을 정의합니다.
**Markdown 형식**으로 작성하여 쉽게 읽고 수정할 수 있습니다.

---

## 1. 필수 규정 (오류)

### 기안 끝 표기
- **규칙**: 기안의 끝은 마침표 다음 **두 칸을 띄우고** '끝.'을 작성합니다
- **잘못된 예시**: `...합니다. 끝.` 또는 `...합니다.끝.`
- **올바른 예시**: `...합니다.  끝.` (마침표 + 공백 2칸 + 끝.)
- **자동수정**: 가능

### 붙임 표기
- **규칙**: 첨부서류가 있을 경우 '붙임:' 또는 '붙임 :'으로 명시하고 목록 작성
- **잘못된 예시**: `붙임 1. 예산서`
- **올바른 예시**: `붙임: 1. 예산서` 또는 `붙임 : 1. 예산서`
- **자동수정**: 가능

## 2. 형식 규정 (경고)

### 날짜 형식 통일
- **규칙**: 날짜는 `YYYY.MM.DD` 또는 `YYYY년 MM월 DD일` 형식으로 작성
- **잘못된 예시**:
  - `2024-11-03` (하이픈 사용)
  - `2024/11/03` (슬래시 사용)
  - `24.11.03` (2자리 연도)
- **올바른 예시**:
  - `2024.11.03`
  - `2024년 11월 3일`
- **자동수정**: 가능

### 경어체 통일
- **규칙**: 문서 전체에서 '~합니다', '~입니다' 등 경어체를 일관되게 사용
- **주의사항**:
  - 한 문장 내에서 경어체와 평서체 혼용 금지
  - 예: "검토하고 승인이 필요합니다" (X) → "검토하며 승인이 필요합니다" (O)

### 제목 형식
- **규칙**: 제목은 간결하고 명확하게 작성하며, 특수문자 사용 지양
- **주의사항**: 느낌표(!), 물음표(?) 등 특수문자 사용 자제

## 3. 공문서 용어 (경고)

### 구어체 → 문어체 변환
다음 구어체 표현을 공식 문서 용어로 변경해야 합니다:

| 구어체 (잘못됨) | 문어체 (올바름) | 설명 |
|---------------|----------------|------|
| 하지만 | 그러나 | 접속사 |
| 근데 | 그런데 | 접속사 |
| 그래서 | 따라서 | 인과 관계 |
| 왜냐하면 | 이는 | 이유 설명 |
| 때문에 | 으로 인하여 | 원인 |

**자동수정**: 가능

### 공식 행정 용어
| 비표준 | 표준 | 용도 |
|--------|------|------|
| ~할려고 | ~하려고 | 의도 표현 |
| ~할꺼야 | ~할 것입니다 | 미래 |
| 바래요 | 바라요 | 희망 표현 |

## 4. 맞춤법 (경고)

### 흔한 맞춤법 오류

| 틀린 표현 | 올바른 표현 | 설명 |
|----------|-------------|------|
| 않됩니다 | 안 됩니다 | '안'은 띄어쓰기 |
| 않됨 | 안 됨 | '안'은 띄어쓰기 |
| 않된 | 안 된 | '안'은 띄어쓰기 |
| 안되 | 안 돼 | '안'은 띄어쓰기, '되'는 '돼' |
| 되어지다 | 되다 | 이중 피동 금지 |
| 이루어지다 | 이루다 | 의미 중복 |
| 갖이 | 같이 | 맞춤법 |
| 갔이 | 같이 | 맞춤법 |
| ~하시기바랍니다 | ~하시기 바랍니다 | 띄어쓰기 |
| 바램 | 바람 | 명사형 |
| 어떻해 | 어떻게 해 | 줄임말 금지 |

**자동수정**: 가능

## 5. 서식 규정 (정보)

### 문단 구분
- 문단은 명확하게 구분
- 적절한 들여쓰기 사용
- 빈 줄로 문단 구분

### 불필요한 공백 제거
- **규칙**: 연속된 공백은 하나로 통일 (단, '끝.' 표기의 두 칸 제외)
- **예시**: `세부  내용` → `세부 내용`
- **자동수정**: 가능

### 숫자 표기
- 큰 금액: 아라비아 숫자 사용 (예: 50,000,000원)
- 작은 수량: 한글 또는 아라비아 숫자 (예: 1부, 한 부)
- **일관성 유지**가 중요
